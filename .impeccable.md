## Design Context

### Users
本项目的主要使用场景是课程演示。

- 直接使用者是讲师、助教或演示者，需要在有限时间内快速讲清楚知识图谱如何完成职业推荐、差距分析和行动模拟。
- 观看者是课堂上的学生或听众，他们通常不是来做长时间深度操作，而是跟随演示理解系统逻辑、结果解释和路径推演。
- 这个界面首先服务“讲解效率”和“演示流畅度”，其次才是高密度专家操作。

### Brand Personality
品牌气质以“严谨、易用、便捷”为核心。

- 严谨：推荐逻辑应显得可信、可解释、可追溯，避免花哨科技感和过度营销化表达。
- 易用：信息结构要一眼能懂，讲师在演示时不需要解释界面本身。
- 便捷：关键动作路径要短，状态切换要清楚，适合课堂中的连续演示和快速回放。

界面应传达的情绪是：可信、清晰、从容，而不是兴奋、炫技或压迫感。

### Aesthetic Direction
整体视觉方向保留学术气质，但不是传统政务风或呆板后台风，而是“研究工作台 / 课程讲义 / 可视化实验台”的融合。

- 主题固定为浅色，优先适配投影、教室显示器和白天演示环境。
- 视觉参考方向：
  - 学术海报与研究图表的清晰层级
  - 高质量数据新闻的编辑感排版
  - 实验工作台式的信息分区与推演感
- 反向约束：
  - 不要做成消费级求职 App
  - 不要做成发光深色科技大屏
  - 不要做成通用 BI 仪表盘或“卡片堆叠后台”

应当让人记住的是：这是一个能把推荐过程“讲明白”的知识图谱工作台，而不是只给结论的列表页。

### Design Principles
1. 先讲逻辑，再讲结果。所有视觉层级都要服务“输入 -> 推理 -> 结果/差距 -> 模拟”的讲解链路。
2. 演示优先。高频动作必须醒目、顺手、低认知负担，适合在课堂中连续点击和切换。
3. 学术而不沉闷。保留研究型产品的可信度与秩序感，但通过排版、配色和节奏避免僵硬。
4. 解释性强于装饰性。装饰元素只能强化图谱、路径、证据与溯源，不得喧宾夺主。
5. 浅色环境下保持投影可读性。对比度、字号、分区边界和状态色都要优先为教室演示服务。
